# Smoke Test Checklist (P0, 10~30분)

대상: 제출/배포 직전 수동 확인  
환경: 기본은 staging 또는 production read-only 계정  
원칙: 실패 항목 1개라도 있으면 배포 보류

## 1) 인증 진입 (3분)
- [ ] `/login` 진입 시 폼 렌더 정상
- [ ] 이메일 로그인 성공 후 `/app/insights` 진입
- [ ] 로그아웃 클릭 시 `/login` 복귀
- 기대 결과:
  - 인증 루프 무한 리다이렉트 없음
  - 첫 화면 로딩 스피너가 장시간 지속되지 않음

## 2) Daily Flow 저장 핵심 경로 (5~8분)
- [ ] `/app/daily-flow` 진입
- [ ] 일기 10자 이상 입력 후 `AI 분석하기`
- [ ] confirm 화면 진입 확인
- [ ] 시간 미확정 항목에 시간대 칩 1개 선택
- [ ] `확인 & 저장` 클릭
- 기대 결과:
  - 저장 실패 배너 없이 `저장 완료` 단계 표시
  - Supabase 관련 일반 오류 문구가 상시 노출되지 않음

## 3) 분석/리포트 경로 (4~6분)
- [ ] Daily Flow에서 `AI 분석` 클릭
- [ ] `/app/reports/[date]` 이동
- [ ] `오늘의 요약`, `내일의 추천 흐름` 카드 렌더
- [ ] 리포트 없는 날짜 접근 시 404가 에러가 아닌 empty state로 표시
- 기대 결과:
  - 분석 실패 시 재시도 가능 메시지
  - 빈 상태/에러 상태 구분 명확

## 4) 설정 패널 핵심 경로 (4~6분)
- [ ] 인사이트 하단 톱니(FAB) 클릭 시 설정 모달 오픈
- [ ] `개인설정` 탭에서 4개 필수값 저장
- [ ] `데이터 제어` 탭에서 초기화 확인창 동작(실운영에서는 테스트 계정만)
- [ ] `계정` 탭에서 플랜/이메일 표시
- 기대 결과:
  - `/app/preferences` 직접 진입해도 설정 모달 프로필 탭으로 유도
  - 저장/삭제 후 성공 피드백 표시

## 5) 회원탈퇴 경로 (테스트 계정 한정, 3~5분)
- [ ] `설정 > 계정 > 회원탈퇴` 클릭
- [ ] 확인 후 `/login?deleted=1` 이동
- 기대 결과:
  - API 500/타임아웃 없이 완료
  - 동일 계정 재사용 시 정책대로 동작

## 6) 반응형 최소 확인 (2~3분)
- [ ] 모바일(390), 태블릿(768), 데스크톱(1440)에서 겹침/클리핑 없음
- [ ] 하단 탭 + 설정 FAB + 모달 z-index 충돌 없음

## 7) 체크포인트
- [ ] 콘솔에 치명 에러 없음(일시적 warning 제외)
- [ ] 네트워크에서 핵심 API가 2xx/의도된 404(빈 상태)로 처리
