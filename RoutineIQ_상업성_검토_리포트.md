# RoutineIQ 상업성 검토 리포트

## 1. 현재 상태 요약

RoutineIQ는 Next.js(프론트) + FastAPI(백엔드) + Supabase(DB/인증) + OpenAI(AI분석) + Stripe(결제) 스택으로 구성된 해커톤 수준 MVP입니다. 핵심 플로우(기록 → AI 분석 → 리포트)는 동작하지만, 상용 서비스로 가려면 아래의 수정/추가/삭제가 필요합니다.

---

## 2. 현재 잘 되어 있는 부분 (유지)

- **아키텍처 설계**: Supabase RLS + API 이중 보안, 서비스 역할 분리가 체계적
- **AI 리포트 스키마**: OpenAI Structured Outputs + Pydantic 검증으로 일관된 품질
- **결제 통합**: Stripe 웹훅 서명 검증, 결제 미설정 시 graceful degradation
- **UI 디자인 톤**: Space Grotesk + Fraunces 폰트 조합, 크림색 배경의 따뜻한 톤
- **데이터 프라이버시**: 최소 PII 로깅, 내보내기/삭제 기능 제공

## 3. 반드시 수정해야 할 것 (MODIFY)

### 3-1. 랜딩 페이지 — 가치 전달 실패

**현재 문제**: "beta (hackathon build)" 태그가 노출되고, 예시 카드 하나만 보여줌. 경쟁 앱과 비교했을 때 RoutineIQ만의 차별점("AI가 실패 패턴을 분석해 루틴을 자동 개선")이 전혀 드러나지 않음.

**수정 방향**:
- "hackathon build" 제거 → "AI-Powered Routine Optimizer" 등으로 교체
- Hero 섹션에 3단계 플로우 시각화: "기록(1분) → AI 분석 → 맞춤 루틴"
- 실제 AI 리포트 샘플 스크린샷 or 애니메이션 데모
- Social proof 섹션 (얼리 유저 후기, 사용 통계)
- 경쟁사 대비 차별점 섹션 ("다른 습관앱은 기록만, RoutineIQ는 실패원인 분석 + 자동개선")

### 3-2. Daily Flow 입력 UX — 진입장벽이 높음

**현재 문제**: 시작/종료 시간을 수동 입력해야 하고, Energy/Focus를 1-5 숫자로 직접 타이핑. 모바일에서 12개 입력필드를 채우기는 현실적으로 불가능.

**수정 방향**:
- 시간 입력을 **드래그 타임라인** 또는 **30분 단위 슬롯 선택**으로 변경
- Energy/Focus를 **이모지 슬라이더**(😴→🔥) 또는 **탭 선택 버튼**으로 교체
- **자동 시간 감지**: "지금 시작" 버튼으로 현재 시각 자동 입력
- 시간 겹침/역전 검증 로직 추가 (현재 end < start 허용됨)
- 모바일 우선: 한 블록씩 풀스크린 카드로 입력

### 3-3. Insights 대시보드 — 리포트 없을 때 허전함

**현재 문제**: AI 분석을 안 돌리면 5개 카드가 전부 "No data yet. Analyze to generate." 텍스트만 보임. 첫 방문 사용자가 이탈할 가능성 매우 높음.

**수정 방향**:
- **빈 상태 디자인**: 각 카드에 가이드 일러스트 + "이런 결과가 나옵니다" 샘플 미리보기
- **온보딩 플로우**: 첫 로그인 시 "3분 퀵스타트" 가이드 → 템플릿으로 즉시 기록 → 바로 분석
- **자동 분석 트리거**: 로그 저장 시 "AI 분석도 같이 할까요?" 프롬프트
- Consistency Score를 7일이 아닌 **현재까지의 누적**으로 변경 (첫날부터 100% 가능하게)

### 3-4. 가격 정책 — Free가 너무 제한적, Pro가 약함

**현재**: Free(1회/일, 3일 보관) / Pro($9.99, 10회/일, 30일 보관)

**문제점**:
- Free 유저가 하루 1회 분석 + 3일 보관이면 서비스 가치를 체감하기 어려움
- Pro의 10회/일은 과잉 (누가 하루에 10번 분석하나?)
- Pro와 Free의 리포트 내용 차이가 프론트에서 체감 불가

**수정 방향**:
- **Free**: 3회/주, 7일 보관, 기본 리포트
- **Pro** ($6.99~$8.99): 무제한, 90일 보관, 심화 분석(주간 트렌드, 월간 리포트), 커스텀 루틴 목표
- **Premium** ($14.99): 연간 리뷰, 팀/가족 공유, 외부 앱 연동 (캘린더, 건강앱)

### 3-5. 모바일 UX — 반응형이지만 네이티브 경험 부족

**현재 문제**: 하단 네비게이션은 있지만, 입력 폼이 데스크톱 중심. 터치 타겟이 작고, 스와이프 제스처 없음.

**수정 방향**:
- 입력 폼 전체를 모바일 우선으로 리디자인
- 날짜 이동을 **좌우 스와이프**로
- 버튼 최소 터치 영역 44x44px 보장
- 중장기: React Native 또는 PWA로 전환

### 3-6. 보안/안정성 이슈

**수정 필수 항목**:
- API rate limiting 없음 → IP 기반 + 유저 기반 이중 제한 추가
- Stripe 웹훅 멱등성(idempotency) 미구현 → 중복 처리 방지
- 에러 응답 형식 불일치 → 통일된 에러 스키마
- 입력값 검증 부족 (시간 역전, 에너지 범위 초과 등)
- OpenAI 프롬프트 인젝션 리스크 → 사용자 입력 sanitize 강화

## 4. 제거해야 할 것 (REMOVE)

### 4-1. "hackathon build" 텍스트
- 랜딩 페이지의 beta 배지 제거. 상용 서비스 인상을 줌

### 4-2. 내부 기술 용어 노출
- Billing 페이지의 "Backend-enforced hard limits", "Synced from Stripe via signed webhooks" 등은 개발자 용어
- → "분석 횟수는 자동 관리됩니다", "결제 상태" 같은 사용자 언어로 교체

### 4-3. 과도한 Admin 노출
- Admin 대시보드가 같은 앱 내에 존재하면 보안 리스크
- → 별도 서브도메인(admin.routineiq.com) 또는 별도 배포로 분리

### 4-4. localStorage 기반 최근 활동 저장
- 기기 간 동기화 불가, 데이터 유실 위험
- → Supabase에 유저별 최근 활동 저장으로 이전

### 4-5. 데모/게스트 모드의 과도한 복잡성
- 현재 익명 로그인 → 이메일 전환 플로우가 UX적으로 혼란
- → 데모는 로그인 없이 목업 데이터로 체험하도록 단순화 (DB 저장 없이)

## 5. 반드시 추가해야 할 것 (ADD)

### 5-1. 온보딩 플로우 (최우선)
- 첫 로그인 후 3단계 가이드: ① 오늘 일과 간단 입력 → ② 즉시 AI 분석 → ③ 결과 확인
- 이 과정이 **3분 이내**에 끝나야 함
- "Aha moment"를 첫 5분 안에 만들어야 리텐션 확보

### 5-2. 푸시 알림 / 리마인더
- 매일 저녁 "오늘 하루를 기록하세요" 알림
- 아침 "AI가 설계한 오늘의 루틴을 확인하세요" 알림
- 습관 앱의 핵심 리텐션 도구 — 현재 전혀 없음

### 5-3. 주간/월간 트렌드 리포트
- 현재는 일일 리포트만 존재
- 상용 서비스라면 "이번 주 생산성 트렌드", "월간 성장 곡선" 필수
- Pro 전용 기능으로 차별화 가능

### 5-4. 목표 설정 기능
- "이번 달 목표: 매일 2시간 딥워크" 같은 목표 설정
- AI가 목표 달성률을 리포트에 반영
- 목표 없는 루틴 관리는 방향이 없음

### 5-5. 소셜/공유 기능
- 주간 성취 카드를 이미지로 공유 (인스타, 트위터)
- 바이럴 성장의 핵심 수단
- "나의 이번 주 생산성 점수: 87점 🔥" 같은 공유 카드

### 5-6. 외부 서비스 연동
- Google Calendar 연동 → 일정 자동 입력
- Apple Health / Google Fit → 수면, 운동 데이터 자동 반영
- 수동 입력 의존도를 줄여야 장기 사용 가능

### 5-7. 다국어 지원
- 현재 영어 전용. 한국 시장 타겟이면 한국어 필수
- i18n 프레임워크(next-intl 등) 도입

### 5-8. 이메일 인증 / 비밀번호 재설정
- 현재 없음. 사용자가 비밀번호를 잊으면 계정 접근 불가
- Supabase Auth에 기본 제공되므로 연동만 하면 됨

### 5-9. 에러 바운더리 / 오프라인 대응
- 네트워크 끊김 시 아무 피드백 없음
- 입력 중 데이터 유실 방지를 위한 로컬 임시 저장
- 글로벌 에러 바운더리 컴포넌트

### 5-10. SEO / 마케팅 기반
- 현재 메타태그, OG 이미지 없음
- 블로그/콘텐츠 마케팅 페이지 (루틴 관리 팁 등)
- Google Analytics / Mixpanel 이벤트 트래킹

## 6. 경쟁사 대비 포지셔닝

| 비교 항목 | Habitify | Streaks | Fabulous | **RoutineIQ** |
|---|---|---|---|---|
| 핵심 컨셉 | 습관 체크리스트 | 연속 기록 동기부여 | 행동과학 코칭 | **AI 실패분석 + 루틴 자동설계** |
| AI 활용 | 예측 알림(기본) | 없음 | 가이드 챌린지 | **GPT 기반 심층 분석** |
| 입력 방식 | 체크/미체크 | 체크/미체크 | 가이드 따라가기 | **시간블록 + 에너지/집중도** |
| 차별화 | 다중 플랫폼 | 미니멀 | 행동과학 | **왜 실패했는지 + 대안 제시** |
| 가격 | $3.33/월~ | $4.99 일시불 | $10/월~ | **$6.99~$9.99/월** |

**RoutineIQ의 진짜 차별점**: 다른 앱은 "했다/안했다"를 기록하지만, RoutineIQ는 "왜 무너졌는지"를 분석하고 "언제 어떻게 다시 시도할지"를 AI가 설계해줌. 이 **"실패 분석 → 자동 루틴 재설계"** 루프가 핵심 가치.

## 7. 우선순위 로드맵

### Phase 1 — 즉시 수정 (1~2주)
1. 랜딩 페이지 리뉴얼 (hackathon 제거, 가치 전달 강화)
2. 온보딩 플로우 구현
3. 내부 기술 용어 → 사용자 언어로 전환
4. 입력값 검증 (시간 역전, 범위 초과)
5. API rate limiting 추가
6. 이메일 인증 / 비밀번호 재설정

### Phase 2 — 핵심 기능 보강 (2~4주)
1. Daily Flow 입력 UX 개선 (모바일 우선)
2. 빈 상태 디자인 + 자동 분석 제안
3. 주간 트렌드 리포트 추가
4. 목표 설정 기능
5. 가격 정책 재설계 + 결제 페이지 UX 개선
6. 다국어 지원 (한/영)

### Phase 3 — 성장 기반 (1~2개월)
1. 푸시 알림 / 리마인더
2. 소셜 공유 카드
3. Google Calendar 연동
4. SEO + 콘텐츠 마케팅
5. 이벤트 트래킹 (Mixpanel/GA)
6. PWA 또는 네이티브 앱 전환

### Phase 4 — 확장 (2~3개월)
1. Apple Health / Google Fit 연동
2. 팀/가족 기능
3. Premium 플랜 + 연간 결제
4. Admin 분리 배포
5. 성과 기반 마케팅 (사례 연구, 후기)

## 8. 최종 판단

RoutineIQ의 **핵심 아이디어는 상업적 가치가 있습니다**. "왜 루틴이 무너지는지 AI가 분석하고 자동으로 재설계"하는 컨셉은 기존 습관 앱과 명확히 차별화됩니다. 다만 현재 상태는 해커톤 프로토타입이며, 위에 정리한 수정/추가 작업 없이는 유료 사용자를 확보하기 어렵습니다.

가장 급한 3가지: **① 온보딩 (첫 5분 안에 가치 체감)**, **② 입력 UX 단순화 (1분 내 기록)**, **③ 알림 시스템 (매일 돌아오게 만들기)**.
